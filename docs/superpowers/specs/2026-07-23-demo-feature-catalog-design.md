# TFYSwiftSQLiteKit Demo 功能目录设计

## 目标

将现有一次性文本脚本改造成可交互、可重复运行的 UIKit 功能目录，完整展示 TFYSwiftSQLiteKit 的公开能力。Demo 面向首次接触库的开发者：每个案例可独立运行，输出操作说明、结果和必要的 SQL 轨迹，失败案例明确标记为预期失败。

## 界面

`ViewController` 使用原生 `UITableView` 展示分组案例。每一行包含案例名称和一句说明；点击后进入结果页并立即执行。结果页使用只读等宽字体 `UITextView`，提供重新运行按钮，执行期间显示加载状态。

不引入第三方 UI、Storyboard 或额外架构层。

## 文件职责

- `DemoModels.swift`：保存演示模型、字段定义和迁移回调。
- `DemoCases.swift`：定义案例分组、案例元数据、数据库重置、输出格式及所有执行逻辑。
- `ViewController.swift`：只负责案例列表、导航和结果展示。

## 案例范围

1. Schema 与注解：普通列、主键、自增、索引、唯一索引、默认值、忽略字段、JSON 字段、复合索引及 schema 元数据。
2. 建表与安全迁移：首次建表、增量加列、重复迁移幂等、迁移报告和迁移日志表。
3. 重建迁移：列重命名、自定义转换、重建校验和数据保全。
4. CRUD：单条和批量插入、`insertOrReplace`、主键读取、更新、实例删除及主键删除。
5. 查询：原始条件绑定、动态字段、显式类型字段、AND/OR/NOT、比较、NULL、IN、LIKE、contains、starts、排序、分页、count 和 exists。
6. 事务：提交、回滚和嵌套 savepoint。
7. 多数据库与配置：数据库隔离、路径、WAL、外键、busy timeout、关闭及重新打开。
8. 底层 SQLite：prepare statement、复用 statement、execute、query、scalar、table/index pragma 和 last inserted row ID。
9. 日志与隐私：默认绑定值脱敏、完整绑定日志的开发环境示例、成功和失败 SQL 事件。
10. 错误处理：唯一索引冲突、绑定数量错误、未知字段、无谓词批量删除保护和配置冲突。
11. 性能：小规模读写 benchmark，避免 Demo 启动产生明显卡顿。
12. 数据库维护：关闭连接、删除数据库及 sidecar 文件；破坏性操作只作用于 Demo 专用数据库。

## 数据流与隔离

每个案例声明自己使用的 Demo 数据库名。执行前关闭并删除这些数据库，然后创建所需 schema 和样本数据，因此案例不依赖运行顺序。案例在后台任务中运行，所有 UIKit 更新回到主线程。SQL 日志回调在案例结束时通过 `defer` 恢复为 nil，避免跨案例泄漏状态。

## 输出和错误处理

统一输出标题、步骤、结果及成功标记。预期错误由案例内部捕获并输出“已按预期阻止”；非预期错误交给结果页统一显示。模型和绑定值使用简短摘要，避免输出大块二进制或敏感数据。

## 验证

- 现有 Swift Package 单元测试必须继续通过。
- Xcode iOS Simulator Demo target 必须构建通过。
- 增加一个“运行全部案例”入口，作为最小端到端自检；它顺序执行全部案例并汇总成功/失败数量。
- 不把性能数字作为测试阈值，只验证操作可完成且吞吐量为正数。

## 非目标

- 不构建通用 Demo 框架、依赖注入容器或持久化 UI 状态。
- 不加入网络、云同步、SwiftUI 或第三方依赖。
- 不在 Demo 中操作业务数据库或用户文件。
